# Product Reframing Patterns

## By Product Type

### B2C Products

| Phase | Pattern | Example |
|-------|---------|---------|
| Context | Journey mapping | "유저가 우리 앱을 열기 전에 뭘 하고 있었나요?" |
| Context | Abandonment trigger | "유저가 이탈하는 정확한 순간은?" |
| Context | Comparative behavior | "경쟁 제품에서는 이걸 어떻게 해결하나요?" |
| Context | Frequency/urgency | "이 문제가 하루에 몇 번 발생하나요?" |

### B2B Products

| Phase | Pattern | Example |
|-------|---------|---------|
| Voice | Stakeholder mapping | "이 요청이 누구한테서 왔나요? 실사용자? 구매자? 관리자?" |
| Need | Workflow impact | "이 기능이 업무 프로세스에서 어떤 단계를 바꾸나요?" |
| Context | Organizational | "이 팀의 KPI에 이 문제가 어떤 영향을 주나요?" |
| Context | Compliance/political | "이 요청에 규제나 내부 정치적 이유가 있나요?" |

### Platform/Marketplace

| Phase | Pattern | Example |
|-------|---------|---------|
| Voice | Side identification | "이 요청이 공급자 쪽인가요, 수요자 쪽인가요?" |

## Product-Specific Challenge Mode Adaptations

### Mirror Mode for Products
Focus on "user behavior behind the metric":
- "DAU가 떨어졌다고 하셨는데, 유저가 실제로 뭘 안 하게 된 건가요?"

### Levitt Mode for Products
Apply the Drill→Hole→Picture chain to business metrics:
- "전환율 개선 → 더 많은 결제 → 왜? → 유저가 가치를 빠르게 느끼지 못하는 온보딩 문제"

### Inversion Mode for Products
Question whether the feature should exist at all:
- "이 기능 대신 기존 기능을 제거하면 같은 효과를 낼 수 있을까요?"

## Customer Focused vs Customer Centric Framework

| Approach | Definition | Risk |
|----------|-----------|------|
| **Customer Focused** (고객 집중) | Do what customers SAY | Build a faster horse |
| **Customer Centric** (고객 중심) | Solve what customers EXPERIENCE | Build a car |

### Generating the Comparison

For every Voice Solution, generate a parallel Context Solution and compare:

1. **Cost**: Estimate relative effort (1-5 blocks)
2. **Impact**: Estimate user impact (1-5 blocks)
3. **Risk**: What could go wrong with each approach
4. **Reversibility**: How easy to roll back if wrong

Present the Context Solution as the recommendation, UNLESS Voice Confirmed path is triggered.
